Money / Rails & Plumbing / Seed — Agent‑Centric Switch on Instant Rails

A (Problem): Mobile money (e.g., M‑Pesa) is a walled garden; agents can’t natively interoperate with instant rails or cross‑border corridors.
B (Objective): Make wallets and agents first‑class nodes on national instant systems and low‑cost remittance routes.
Link: An agent‑centric switch maps SIM wallets to instant‑rail aliases, supports offline request‑to‑pay (QR/USSD), and settles to telco float in real time; corridor hubs translate between UPI/PIX/SEPA Instant and mobile wallets.

Core intent: Show that alias‑mapped SIM wallets + offline R2P make agents/wallets behave like first‑class instant‑rail participants domestically and across borders—without changing agent routines.

Load‑Bearing Test & Validation

Show

  • 8‑panel clickthrough: domestic RTP first, then cross‑border.
  • Paper QR + printed settlement ledger.

Validation Reactions

  • Agent ops: “I can accept bank→wallet with existing USSD.”
  • Telco risk: “T+0 to trust removes exposure; fees fit.”
  • Scheme op: “Just another participant/domain with R2P.”
  • Corridor lead: “VPA/PIX ⇄ alias mapping is clear.”
  • Regulator: “Traceable end‑to‑end; KYC anchors stay put.”

Disqualifiers

  • Agent needs new hardware/smartphone
  • No T+0 settlement to telco trust
  • Alias can’t piggyback scheme directory
  • Cross‑border compliance requires offshore consumer funds

Why This is a Box of Dirt

Mocked screens, paper artifacts, and prefilled ledgers test the mapping‑and‑settlement idea without integrations. Minimal (1 domestic flow, 1 corridor, 1 failure), disposable (adjust fees/settlement model if any actor objects), and growth‑inviting (dynamic QR, POS plugins, broader alias domains, merchant payouts, tiered KYC, more corridors). The box proves the smallest claim: agents + wallets act like scheme participants via alias mapping, offline R2P, and T+0 settlement—before any infrastructure is built.
